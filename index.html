Program Help

Navigation

Views

There are three primary views: data table, graph, and help. Use the buttons at the bottom of each page to navigate between these different views.

  • The data table view is where you can enter coordinate pairs or data items that you wish to use for a graph or regression.
  • The graph view is where you can view and edit your graph and its associated properties.
  • The help view (where you currently are) provides information about this application's use.

Controls

If you want to know what any button does, simply hover your mouse over it for a short description.

If you a have a cell of the data table selected there are a number of key operations that can be used to navigate without the mouse:

  • Pressing Tab will move to the next element of the table: right then down.
  • Pressing Shift-Tab will move to the previous element of the table: left then up.
  • Pressing Enter will jump to the next row in the selected column.
  • Presstig Shift-Enter will jump to the previous row in the selected column.

Unmodified titles have the ability to auto-generate based on your labels when written in the form Label (unit). For example, an x label of Time (s) and a y label of Distance (m) would result in a title of Distance v. Time.

Printing

To print a graph, simply save it as an image and use the print functionality on your devices. For best results use the Landscape printing orientation and (on Chromebooks) uncheck the "Headers and footers" option.

Display

The program is intended to be run in the full screen view of a chromebook. Viewing it on lower resolutions may lead to portions of the page not being displayed or inaccessible.

Special characters

Chromebooks

If you are on a Chromebook and want to type the squared or cubed symbols you can change your keyboard to be US International and use your right Alt key along with either 2 or 3.

Desktops

Squared and cubed symbols can be typed by pressing Alt-1-7-8 and Alt-1-7-9, respectively.

Issues

There are a number of situations that will cause problems with your graph:

  1. Always make sure you have the same number of data items in both your X and your Y columns.
  2. Make sure that you haven't skipped any rows of the data table (any data entered after a blank cell is ignored).
  3. Ensure that you don't have any non-number characters (including spaces and enters) inside of your data cells.

If you have any of these problems your graph may not load or will behave expectedly.
